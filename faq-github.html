Pingendo for Github - FAQ

What is the problem Pingendo solves ?

Stakeholders, copywriters, designers, customers often sharing psd, screenshots, mockups, docs but the last version of a work in progress webpage.
Developer is than asked to take everything online, even easy modification or unclear requests.
Wait a moment, developers are not the scarcer resource?
Why do not involve developers only with the 10% code and leave the 90% content iteration to the marketing and communication team?

What is the solution?

Design in the browser.
Everyone working on the same static and plain HTML/CSS source that will evolve from early prototype to production ready.
Designers, marketing and communication freely iterate and experiment on content, UI and UX.
Developers focusing on coding and on managing on the revision platform Github everything and everybody . 

                         

How the solution works?

Once logged, users will be able to browse HTML pages in Github repos, make their changes, proposing revision (pull requests) to the project owners without even knowing what Github or Bootstrap are.

Everyone contributes according to its own role and skill. Basic users will update text and images, while advanced users will quickly create entire webpages. Developers can go inside the vanilla Bootsrap source. 

Messed up? Developers has all the power of Github to fix it.


                         

Who benefits of Pingendo ?


Developers save time by working on code only and on clearer issues.
Web agencies will reduce presentations and will benefit of a smoother workgroup iteration.
Customers better understand living webpages than static mockups.
Marketing and communication will enjoy an unseen independence.
Pingendo is also a great tool to learn HTML/CSS. 

So, how to start ?

Just point to  v4.pingendo.com/github.html

Can i use it on private repositories ?

Not yet, we are working on it. 

How much does it costs?

Not yet defined, what's your idea to be a deal for both you and us? 

Do I have to grant Pingendo access to all my repositories?

Unfortunately Github does not yet features granular control access through API. Pingendo does not store any of your password or giulio scrive. Anyway, if you feel safer, please create a user with only the repositories you would like to connect to Pingendo.
