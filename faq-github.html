Pingendo for Github - FAQ

What is the problem Pingendo solves ?

Back and forth of psd, screenshot, mockups, and every sort of offline docs are today the backbone of teams iteration during the creation of static wepages.
Stakeholders approvals and contribution by copywriters, designers, customers or other resources, mostly happen offline.
Developer is than required to take everything online, even easy modification or unclear requests.
Wait a moment, developers are not the scarcer resource?
Why do not involve developers only with the 10% code and leave the 90% content iteration to the marketing and communication team?

What is the solution?

Design in the browser.
Everyone working on the same static HTML/CSS source that will evolve from early prototype to production ready.
Developers focused on code iterations and marketing and communication freely iterate and experiment on content, UI and UX. 
Developers manages everything and everybody on the revision platform Github. 

                         

How the solution works?

Once logged, users will be able to browse HTML pages in Github repos, make their changes, proposing revision (pull requests) to the project owners without even knowing what Github or Bootstrap are.

Everyone contributes according to its role and skill. Basic users will update text and images, while advanced users will quickly create entire webpages. Learners will push boundaries forward day after day.

Messed up? Developers has all the power of Github to fix it.


                         

Who benefits of Pingendo ?


  Developers save time by working on code only and on clearer issues.
Web agencies will reduce presentations and will benefit of a smoother workgroup iteration.
Customers better understand living webpages than static mockups.
Marketing and communication will enjoy an unseen independence.

How does it work ?

Pingendo for Github is a web app that simplify github.com usage. Once logged, users will be able to browse HTML pages in Github repos, make their changes, submitting them as pull requests without even knowing what Github or Bootstrap are.

Can i use on private repositories ?

Not yet, we are working on. 

How much does it costs ?

Pingendo for Github is free during beta period. Pricing will be per connected Github repository.

So, how to start ?

Just point to  v4.pingendo.com/github.html
