Pingendo for Github - FAQ

The problem

In the process of static webpages creation, the workgroup mainly shares many kind of offline documents such as psd, screenshots, mockups, docs but the last version of a work in progress webpage.
Developer is than asked to take everything online, even easy modification or unclear requests.
Wait a moment, developers are not the scarcer resource?
Why do not 

Our solution

Design in the browser.
Designers, marketing and communication freely iterate and experiment on content, UI and UX. 
Developers code and manage everything and everybody on the revision platform Github.  
Everyone working on the same static and plain HTML/CSS online source that will evolve from early prototype to production ready.

How does Pingendo works

Once logged, users browse HTML pages in Github repositories, proposing revision (pull requests) to the project owners without even knowing what Github or Bootstrap are.

Everyone contributes according to its own role and skill. A common use case might be: Basic users update text and images, advanced users quickly create entire webpages and developers add custom behaviour to the vanilla Bootstrap source.

Got stuck? Your problem is a click away from your developer. Messed up? Developers has all the power of Github to fix it.

Who benefits of Pingendo ?


Developers will save time by working on code only and on clearer issues.
Web agencies will reduce presentations and will benefit of a smoother workgroup iteration.
Customers will better understand living webpages than static mockups.
Marketing and communication will enjoy an unseen independence.
Pingendo is also a great tool to learn HTML/CSS.

Can i use it on private repositories ?

Not yet, we are working on it. 

Do I have to grant Pingendo access to all my repositories?

Unfortunately Github does not yet features granular control access through API. Pingendo does not store any of your password or giulio scrive. Anyway, if you feel safer, please create a user with only the repositories you would like to connect to Pingendo.

I'm not a developer but I can create webpages with Pingendo. Do I need a developer?

You are probably more independent, but a reachable developer is a safe choice in case you get stuck.
Please note that Pingendo works on plain HTML/CSS files in order to be open to all web technologies.
Without a developers you'll miss most of the web features. 

What are the benefits of using Pingendo on Bootstrap 4? 

x Pingendo altre faq Giulio

Pingendo is built on top of Bootstrap to achive this:
1) speed up prototyping for developers
2) Simplify prototyping so that non-dev can be more independent from developers
3) Expose only basic styling in order to keep things simple and keep non-dev in the loop
4) Generate plain HTML/CSS so that dev can style every pixel.
