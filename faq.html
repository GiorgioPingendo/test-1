Developers and non developers, what's different?

Github collaboration

Pingendo's collaboration is built on top of Github. Github knowledges are required to set up a collaboration project. 

Editor

The editor features dev and non-dev features. Users can choose to remain on the surface or to go deep according to everyone's role and skill. The WYSIWYG editor permits to all contributors to perceive real-time the effect of their modification. Some use cases:

  • Copywriters edit texts.
  • Photographers update links to photos.
  • Designers edit lay-out or styles.
  • SEO specialists add page title or description, revise texts and more... 
  • Marketers act where he/she can and distribute tasks.
  • Stakeholders can be main contributors or just be set "read only" and approve the job at required stages.
  • Developers focus on issues the above people can't. e.g. effects, custom responsive rules, deep styling...

What is one of the possible use cases?

  • Developer set up the project in Github and invites the project manager as Github collaborator  
  • The project manager (PM) has a rough idea of what HTML is, so drags some sections to create a rough prototype about what's needed. 
  • PM adds some contents and the related photos.

  • PM asks to designer to add some CSS to improve esthetic.
  • Designer asks to developer some help on CSS.

  • PM tests the online page
  • PM asks to dev to automatically push the master online.

What is a "Pull request"

It's just a copy of the original document (master) where you can add your contribution. it can:

  • Be closed (discarded) by the user who created it.
  • Be merged (accepted) by the project owner
  • Be closed (discarded) by the project owner
  • Every single file in the pull request can be compared to the master by any user.

How to create pull requests efficiently?

To help the project owner to revise your job and easily accept your contribution please remind to:

  • Give them a title related to the modification
  • Create a pull request for each different set of modification
  • Keep them small, with few modification each
  • Comment on each pull request

How to manage contributors efficiently?

To help contributors to create in the required direction please remind to:

  • Revise contributions soon after 
  • Create a pull request for each different set of modification
  • Keep them small, with few modification each
  • Comment on each pull request

What are the different roles?

  • Owner/Developer --> Usually the developer who has full control of the project and the users involved
  • Collaborator/Project manager--> Usually the marketer. He/she will manage the project by accepting/discarding pull requests and by distributing tasks.
  • User --> They can only make contribution by creating pull requests
  • Read only user --> They get only a link to the webpage without the possibility of contribution.

Do users have to signup to Github to submit pull requests?

Yes, because Github manages users and their actions. Don't worry, signing up to Github is just as easy as any other service and no more actions are required on Github.
